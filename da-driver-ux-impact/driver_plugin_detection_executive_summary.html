外掛司機候選識別與平台影響分析

BQ / Ops / T&S

1Driver 動機

數據反映的 driver 行為很清楚：他們不是單純想接更快，而是想少接低價、低效率、低確定性的單，把時間留給高加成、高單價的單。

其核心優化目標不是「接單速度」，而是 每小時收入。

2核心結論

BQ 分析顯示，外掛司機與正常司機的差異不在於「搶單有多快」，而在於是否持續利用 Release 推高加成、重搶同一筆訂單，並把短期加價收益建立在平台完成率下降、用戶等待變長與後續流失之上。

定義 公式 C

3這件事對平台的影響

這不是單純的接單效率問題，而是收益結構被扭曲。net_impact_hkd 說明

  • 短期：單筆訂單附加費與平台佣金可能上升
  • 中期：完成率下降，重下單與等待時間上升
  • 長期：用戶流失增加，最後讓 net_impact_hkd 轉負
簡單說，平台表面上可能看到「單筆多賺一點」，但整體上承受的是「品質下降、補償變多、客戶後續不再回來」。

4外掛候選判斷框架

外掛候選不是看單一指標，而是依序看四件事： 是否進池、證據分數是否足夠、是否已對平台造成實際傷害、以及是否通過人工覆核。

外掛識別信號

BQ 已經顯示，接單速度本身無法區分外掛。近 30 天實查中，最快接單的一批司機完全沒有 Release 記錄。 BQ 證據

具有區分度的連續行為包括：

  • release_rate 異常偏高
  • 放棄後重搶同一筆單
  • 重搶後附加費高於第一次
  • Grace Drop / Driver cancel 異常偏高且集中低價單
  • 高價單保留、低價單放棄

其中 Grace Drop 並非只出現在單一訂單類型，而是 OD + Booking 都會發生； 近 30 天實查顯示，量體以 OD 為主，但 Same Day 與 Next Day 亦可見同類事件， 因此不能把它當成只屬於某一軌的現象。

判斷重點不在接單速度，而在接單後是否出現反覆放棄、等待加價、再接回訂單的操縱循環。

優先關注場景

優先覆核對象並非所有高棄單率司機，而是同時落在以下組合的場景：

  • 高加成貨物
  • 稀疏競爭池
  • 傍晚 / 深夜供給缺口

尤其是機場趕班、搬遷等高壓、高附加費場景，最容易出現「棄單 → 等加價 → 再接」。 高風險場景

非外掛優先解讀場景

以下兩類場景即使出現高棄單率，也不應直接解讀為外掛：

  • 早晨高加成自然搶單
  • 午間低加成、供給本來就不足

前者較接近正常市場搶單行為，後者主要反映供給不足。

分析目標

本分析的目標不是識別接單速度快的司機，而是識別反覆利用 Release 推高加成、重搶同單，並最終對平台造成淨傷害的司機。
